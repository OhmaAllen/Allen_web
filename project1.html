Developing AI Advisor - DUKIES

📅 Duration: May 2024 - Present

📍 Duke University

  • 🚀 Built an AI-powered academic advising system using Retrieval-Augmented Generation (RAG), leveraging retriever and reranker models.
  • 🧠 Integrated and fine-tuned Qwen2.5 LLM with LoRA to improve accuracy and relevance in academic guidance for students.
  • 💻 Designed a Streamlit front-end for an interactive and user-friendly experience.
  • ⚡ Developed a robust FastAPI back-end for efficient handling of concurrent requests.
  • 📊 Curated and cleaned Duke University’s course data, forming specialized datasets to enhance model fine-tuning and vector database building.

Project Demonstration


By web scraw the data available at DUKE official website, we build our database for the whole CS and enhance courses offered by Duke as well as the professor's information.

We use these info to fine-tune our LLM with LoRA configuration

We vectorize our database using Milvus and build a two step RAG system:
1. Retrieve top-k relevant course and professor information using the retriever model. 2. Rerank the retrieved information to ensure the most relevant data is presented. 3. Generate personalized academic advice using the fine-tuned Qwen2.5 LLM.

We use FastAPI to handle the concurrent requests and Streamlit to provide a user-friendly interface.
